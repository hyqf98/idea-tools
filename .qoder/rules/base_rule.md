# 工程描述

当前项目是根据 IDEA + Java + gradle 进行构建开发的插件

# 框架描述

- 测试版本：idea-IC 2024.3.x - 2025.2.x版本
- JDK：17
- Gradle：8.12.1
- org.jetbrains.intellij.platform：2.7.0
- com.intellij.java
- lombok

在开始开发功能时你必须先充分阅读 **org.jetbrains.intellij.platform** 、**com.intellij.java** 等API的使用
能够复用对应插件的功能就复用对应插件的功能，不得自己随意编写，如果没有对应的API你再考虑自己实现

# 开发要求

请注意在开发代码工程中的风格要求，如下：

- 新增的任何配置都需要添加对应的配置说明和用途，用于代码理解
- 代码开发要求每个类、方法、字段都需要使用标注的Java Doc来进行描述方法的功能和用处
- 代码风格你需要优先查看JDK17的语法风格来进行标准化的代码开发
- 字段、方法、类的命名一定要规范，需要做到见名知意的地步
- 所有的空、字符串、集合等判断优先使用hutool提供的工具方法来进行判断
- 所有对集合的操作，优先使用Stream流式API来进行操作
- 使用Optional等工具类来帮助减少if else的代码书写
- 如果类型分支比较多，优先使用switch case语法
- 所有的工具类方法，如字符串转换、驼峰划分等，优先查看hutool是否提供工具来使用，否则就创建公共的工具类方法来进行管理
- 工具类方法的抽取一定要满足通用性
- 在新增功能时需要考虑对应的扩展性能，例如：优先抽取接口，接口下面抽取Abstract类来管理通用方法，功能进行实现和扩展即可
- 同一个类，方法与方法内部进行交互时，如果参数过多了，优先使用jdk17的recard类来进行管理和传递
- 创建类时，使用lombok框架；对象进行构建时使用@Builder的方式构建，包括get、set等方法均使用@Data替换
- 充分理解需求并且考虑是否引入设计模式、异步等模式来提高插件的性能（注意线程池等代码复用）

# 注意事项

- 当用户没有明确指示需要生成对应的文档或者单元测试时，禁止生成文档和测试类
- 注意import导入包时不能使用 import xx.* 的样式进行全量导入
- 如果是spring框架，引入service，优先使用@lombok的注解，并且通过 private final
